Async/Await
2019-01-19 13:08   folder FrontEnd label JavaScript

Usage

使用 Async/Await 让异步写起来更简单, 感觉像是在写同步代码一样:

async function logFetch(url){
  try{
    console.log(1)
    const response = await fetch(url)
    console.log( await response.text())
    console.log(2)
  }
  catch(err){
    console.log('fetch failed, ', err)
  }
}

logFetch('https://jsonplaceholder.typicode.com/todos/1')

结果如下:

logFetch 函数是这样执行的:

  • 先在控制台输出 1
  • 执行 fetch 函数
  • 等待返回的结果并打印结果到控制台
  • 在控制台输出 2

那直接去掉 Async/Await 呢?

function logFetch(url){
  try{
    console.log(1)
    const response = fetch(url)
    console.log(response.text())
    console.log(2)
  }
  catch(err){
    console.log('fetch failed, ', err)
  }
}

logFetch('https://jsonplaceholder.typicode.com/todos/1')

结果如下:

logFetch 函数是这样执行的:

  • 先在控制台输出 1
  • 执行 fetch 函数
  • 这里不一样了!! 由于浏览器不会等待 fetch() 函数的结果, 直接打印 response.text() 就报错了
