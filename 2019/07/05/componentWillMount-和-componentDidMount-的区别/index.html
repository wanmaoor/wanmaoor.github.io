componentWillMount 和 componentDidMount 的区别
2019-07-05 16:39   folder FrontEnd label React

执行顺序

​ 假如有两个相同的组件, 在各自的compopnentWillMount和componentDidMount两个生命周期函数里打一个控制台函数.

componentWillMount() {
    console.log('enter componentWillMount ' + this.props.caption);
  }

  componentDidMount() {
    console.log('enter componentDidMount ' + this.props.caption);
  }

     结果并不会与预期相同: 

enter componentWillMount First 
enter render First 
enter componentWillMount Second
enter render Second
enter componentDidMount First 
enter componentDidMount Second

​ 可以看到componentWillMount总是在render函数被调用前调用. componentDidMount是在render函数调用完后才开始调用.由于render函数不参与DOM的渲染,而是负责向react库返回JSX对象.所以只有等到react库把render函数返回的JSX对象解析完后才会开始调用各个组件的componentDidMount生命周期函数

执行环境

​ componentWillMount既可以执行在服务端,也可以执行在客户端(即浏览器环境)

​ componentDidMount只能在客户端执行, 原因也很简单: 服务器上没有浏览器环境,不能渲染DOM树.

​ 所以componentDidMount是只能在浏览器执行.也给开发者很好的地方去做只有浏览器才做的事情,比如Ajax请求.
